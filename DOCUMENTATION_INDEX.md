## üé® Documentation Standards

### **File Naming**

- Use `UPPER_SNAKE_CASE.md` for root-level docs
- Use `kebab-case.md` for nested docs
- Use descriptive names (avoid `doc1.md`, `guide.md`)

### **Markdown Style**

- Use ATX-style headers (`#`, `##`, `###`)
- Include table of contents for long docs
- Use code blocks with language tags
- Include emoji for visual clarity (sparingly)
- Keep lines under 120 characters

### **Document Structure**

```markdown
# Title

> Subtitle / Metadata
> Last Updated: YYYY-MM-DD

---

## Table of Contents (if >200 lines)

## Introduction

## Main Content

## Conclusion / Summary

---

**Document Metadata**
```
